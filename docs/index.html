single-cell RNA-seq analysis pipeline

This is the Seurat workflow applied to single-cell RNA sequencing data from multiple bladder cancer samples using the Seurat package in R. I have added some details in each step that will help to perform this pipeline from scractch. The goal is to read raw scRNA data, create a seurat object, perform quality control to filter out poor quality cells, merge and integrate multiple samples to correct for batch effect and lastly annotate cell clusters. The pipeline also contain the doublet finder (doublet cells) and ambiant RNA removal steps.

I have used four files, sample1 - sample4, in this workflow and processed each sample individually for more clarity.

Load Raw Expression Matrices

10X genomics is the most widely used single cell RNA sequencing platform. There are various RNA-seq quantification tools used which transform raw sequencing reads into a count matrix (genes x cells). Cell Ranger is the official 10X pipeline which uses STAR aligner to align and quantify raw reads into a count matrix. The CellRanger pipeline output scRNA raw data results in three files which together define the count matrix. These three files for each sample generated by the Cell Ranger pipeline includes:

  • matrix.mtx: the raw count matrix

  • features.tsv: gene names

  • barcodes.tsv: cell barcodes .

We will use the feature, barcode, matrix files downloaded from a GEO study GSE135337, processed by the CellRAnger pipeline in this workflow. The code to read these files is demonstrated here. ReadMtx function will read these three files for each sample and convert into a single sparse matrix. Consider this count matrix of feature, barcode, matrix as a count matrix in bulk RNA-seq data. The rows are genes and columns are cell barcodes or cells. We will then convert this sparse count matrix into a seurat object.

# Replace these paths with your own where your matrix, feature,barcode files for each sample are placed in your local computer

 sample1 <- ReadMtx(mtx = "path/to/sample1/matrix.mtx", features = "path/to/sample1/features.tsv",
  cells = "path/to/sample1/barcodes.tsv")

 sample2 <- ReadMtx(mtx = "path/to/sample2/matrix.mtx",features = "path/to/sample2/features.tsv",
  cells = "path/to/sample2/barcodes.tsv")

sample3 <- ReadMtx(mtx = "path/to/sample3/matrix.mtx", features = "path/to/sample3/features.tsv",
  cells = "path/to/sample3/barcodes.tsv")

sample4 <- ReadMtx(mtx = "path/to/sample4/matrix.mtx", features = "path/to/sample4/features.tsv",
  cells = "path/to/sample4/barcodes.tsv")

After running the above code each sample is transformed into a sparse matrix. I have uploaded the RDS files (carrying sparse matrix for each sample) in this repository.You can download these RDS files for each sample to run this workflow. RDS file is basically how we save any seurat object in R and can later use it again. I have provided the code to save and load RDS in the end. Here I am reading all the four RDS files uploaded in this repository with a for loop.

files <- paste0("sample", 1:4, ".rds")
base_url <- "https://raw.githubusercontent.com/imranS86/single-cell-RNA-seq-pipeline/main/docs/"

for (i in seq_along(files)) {
  temp_file <- tempfile(fileext = ".rds")
  
  download.file(
    paste0(base_url, files[i]),
    temp_file,
    mode = "wb"
  )
  
  assign(
    paste0("sample", i),
    readRDS(temp_file),
    envir = .GlobalEnv
  )
}

Create Seurat Objects

We now create individual Seurat objects for each sample from thier respective sparse matrix. Each object represent a single sample’s scRNA data. We apply basic quality control filter:

  • min.cells = 3: Only keep genes expressed in at least 3 cells.

  • min.features = 200: Only keep cells that have at least 200 detected genes (features).

In addition, we also calculate the percentage of mitrochondrial gene expression extracted by “^MT-” pattern from the data. This will create a new column percent.mt in the metadata that will demonstrate the mitochondrial percentage. Similarly, ribosomal percentage in each cell will be extracted by “^RPS|^RPL” pattern.

  # Sample 1
  sample1 <- CreateSeuratObject(counts = sample1, project = "sample1", min.cells = 3, min.features = 200)
  sample1 <- PercentageFeatureSet(sample1, pattern = "^MT-", col.name = "percent.mt")
  sample1 <- PercentageFeatureSet(sample1, pattern = "^RPS|^RPL", col.name = "percent.ribo")

  # Sample 2
  sample2 <- CreateSeuratObject(counts = sample2, project = "sample2", min.cells = 3, min.features = 200)
  sample2 <- PercentageFeatureSet(sample2, pattern = "^MT-", col.name = "percent.mt")
  sample2 <- PercentageFeatureSet(sample2, pattern = "^RPS|^RPL", col.name = "percent.ribo")

  # Sample 3
  sample3 <- CreateSeuratObject(counts = sample3, project = "sample3", min.cells = 3, min.features = 200)
  sample3 <- PercentageFeatureSet(sample3, pattern = "^MT-", col.name = "percent.mt")
  sample3 <- PercentageFeatureSet(sample3, pattern = "^RPS|^RPL", col.name = "percent.ribo")

  # Sample 4
  sample4 <- CreateSeuratObject(counts = sample4, project = "sample4", min.cells = 3, min.features = 200)
  sample4 <- PercentageFeatureSet(sample4, pattern = "^MT-", col.name = "percent.mt")
  sample4 <- PercentageFeatureSet(sample4, pattern = "^RPS|^RPL", col.name = "percent.ribo")

Quality Control and Filtering

For quality control explore multiple metrics together. Don’t remove cells based on just one factor such as gene counts or UMI counts. Instead explore multiple metrics together including UMI counts, mitochondrial percentage, ribosomal percentage, number of detected genes. You might filter a dormant cell considering it as a dying cell. A dying cell has a high mitochondrial percentage and low UMI counts whereas a dormant cell has low UMI count but normal mitochondrial percentage. So we need to consider these factors during QC step.

Doublet removal

During single cell RNA sequencing (a droplet based technology), sometimes more than one cell get capture in a single droplet containing the transcriptome of two cells which badly interfere the analysis so it is handy practice to remove doublets. Here I use scDblFinder package that flags artificial doublet cells.

For doublet detection we will first convert the seurat object into a SingleCellExperiment (as scdblFinder package require input in this format).

Ambiant RNA removal

In single cell RNA sequencing during droplet capture cells are lysed as a result their mRNA spread to nearby cells. This solution referred as “ambient RNA” is added to each cell contaminating the expression profile. For instance you get an immune cell cluster with high epithelial markers, you might call it epithelial cell when really it is just an immune cell with high epithelial genes because of the ambient RNA contamination.

Here we use decontX, a tool for ambient RNA removal. decontX also need a single cell experiment object.

#Extract counts from the merged Seurat Object
raw_counts <- GetAssayData(merged_seurat, slot = "counts")

#Convert into Single Cell Experiment and run decontX 
sce_decontx <- SingleCellExperiment(list(counts= raw_counts))
sce_decontx <- decontX(sce_decontx)

#optional check contamination per cell
contamination <- colData(sce_decontx)$decontX_contamination

#Add decontaminated counts to Seurat
decount_counts <- round(decontXcounts(sce_decontx))
merged_seurat[["decontXcounts"]] <- CreateAssayObject(counts = decount_counts)
#Add contamination to metada
merged_seurat$decontX_contamination <- contamination

#set new default assay to decontaminated counts
DefaultAssay(merged_seurat) <- "decontXcounts"

Seurat standard workflow

Preprocessing for PCA and clustering (Before Integration)

Seurat workflow involves normalizing the raw counts, identifying the highly variable genes (top 2000 by default in each cell), scale the data so that it is comparable across cells and samples.

We first create another seurat object that we will need to run seurat workflow from scratch later. So we will create another seurat object “merged_integrated_seurat” here.

merged_integrated_seurat <- merged_seurat
# Normalize gene expression counts using log-normalization
merged_seurat <- NormalizeData(merged_seurat)

# Identify 2000 highly variable genes (used for downstream steps like PCA) in each cell
merged_seurat <- FindVariableFeatures(merged_seurat)

# Scale and centering the data (zero-mean, unit variance)
merged_seurat <- ScaleData(merged_seurat)

# Run Principal Component Analysis (PCA) to reduce dimensionality
merged_seurat <- RunPCA(merged_seurat)

Determine Optimal Number of Principal Components

Choosing the right number of Principal Components (PCs) is crucial for clustering and integration. Seurat suggest Elbow Plot for this purpose.

# Visual inspection using elbow plot
ElbowPlot(merged_seurat, ndims = 20)

In addition to elbow plot, I use a heuristic method to select PCs taken from Hamid Ghaedi: https://github.com/hamidghaedi/scRNA_seq-analysis. Too few PCs can miss meaningful clusters, too many include noise. We select PCs capturing most variance. You can just use the elbow plot but I also use this heuristic method to get the exact optimal PCs to proceed (but this is optional step).

# Heuristic approach to select PCs
pct <- merged_seurat[["pca"]]@stdev / sum(merged_seurat[["pca"]]@stdev) * 100
cumu <- cumsum(pct)
co1 <- which(cumu > 90 & pct < 5)[1]
co2 <- sort(which((pct[1:length(pct) - 1] - pct[2:length(pct)]) > 0.1), decreasing = TRUE)[1] + 1
pcs <- min(co1, co2)
pcs #return the optimal PCs 
## [1] 15

Integrate samples with Harmony

Different samples might have batch effects due to technical variation (e.g processed on different days, different sequencing machines or processed in different labs). Seurat provides various integration methods such as Harmony, Canonical Correlation Analysis (CCA), RPCA to integrate different samples and reduce technical noise.

I use harmony here. As suggested by Ming Tang that “Harmony is fast, simple and works well for batch correction. It works on PC embeddings and identify cells which should be similar across batches, learns correction factors and adjust embeddings to remove batch effects while preserving real biology. For details about whether you need integration or not for your dataset, read the singlle cell guide by Tommy Tang: https://crazyhottommy.github.io/practical_scRNAseq_analysis_with_R/seurat-workflow.html

Here we will run the seurat standard workflow again to integrate samples using harmony in a single code block using %>% operatior in dplyr R package to minimize batch effect. we will use the “merged_integrated_seurat” object we created above.

merged_integrated_seurat <- merged_integrated_seurat %>%
  NormalizeData(normalization.method = "LogNormalize", scale.factor = 10000) %>%
  FindVariableFeatures(selection.method = "vst", nfeatures = 2000) %>%
  ScaleData() %>%
  RunPCA() %>%
  RunHarmony(group.by.vars = "Patient", dims.use = 1:15) %>%
  RunUMAP(reduction = "harmony", dims = 1:15) %>%
  FindNeighbors(reduction = "harmony", dims = 1:15) %>%
  FindClusters(resolution = 0.4)

Cell Annotation by marker genes

After getting the cell clusters we now need to annotate these clusters. Cell annotation is the trickiest step of single cell RNA workflow as there are hundreds of cell annotation methods so it is very hard to agree which is the most reliable method. Generally the most trustworthy method is manual where you use known marker genes based on literature data to annotate cells.

In addition there are reference based tools such as SingleR where cells are compared based on pre-annotated reference datasets. Senior computational biologists suggest to use both approaches. Use tools to have a general idea and then validate with marker genes instead of merely using a single approach. Here I use the manual marker based approach to annotate cells.

Based on literature data (previous studies and databases) we define the markers for major cell types.

# define marker genes for major cell types
markers <- list(
  "epithelial" = c("EPCAM", "KRT17", "KRT18", "KRT5", "SFN"),
